# Тестовое задание: интеграция с Яндекс.Картами

## Стек: Laravel (бэкенд / API) + Vue 3 (SPA)

### Контекст

Мы развиваем сервис работы с отзывами — сбор, автоответы, виджет отзывов. Одна из ключевых задач сервиса — собирать отзывы с площадок, у которых нет официального API. Это задание — упрощённая модель реальной фичи: подключить карточку организации на Яндекс.Картах и показать её отзывы и рейтинг.

### Что нужно сделать

Небольшое приложение из двух экранов:
1. Авторизация (логин / пароль). Регистрация не нужна — достаточно одного сид-пользователя. Бэкенд на Laravel, SPA-аутентификация через Sanctum.
2. Страница настроек. Поле для вставки ссылки на карточку организации в Яндекс.Картах, валидация ссылки, сохранение в БД. После сохранения — подтягивание данных по этой организации.
3. Вывод данных организации:
  - все отзывы по карточке (на практике Яндекс отдает примерно последние 600 - нужно вытянуть их полностью);
  - постраничная навигация по 50 отзывов на страницу;
  - средний рейтинг компании;
  - точные числа: количество оценок и количество отзывов отдельно
  - Каждый отзыв: автор, дата, текст, оценка.

### Про парсинг (главное)

Официального API у Яндекса нет — данные получаете парсингом, и это центральная часть задания. 

Парсер должен:
- принимать ссылку на любую организацию в Яндекс.Картах;
- доставать все доступные отзывы (ориентир — до ~600), а не только первую выдачу;
- корректно тянуть: средний рейтинг, число оценок, число отзывов.

У Яндекса есть защита от ботов и подгрузка отзывов скриптом по мере прокрутки — справиться с этим и есть суть задачи. Как именно (headless-браузер, разбор внутренних запросов карточки, что-то ещё) — на ваше усмотрение, в README поясните выбор.

### Постраничная навигация
50 отзывов на страницу, переключение страниц без перезагрузки. Подумайте, что грузить за один раз, а что подкачивать: тянуть все ~600 разом по каждому запросу или кэшировать результат парсинга на стороне бэка — решение и его обоснование тоже оцениваем.

### Технические требования
- Логика парсинга и работы с внешним источником вынесена из контроллеров (сервис / класс-обёртка), не «в лоб» в маршруте.
- Валидация входных данных, осмысленная обработка ошибок (страница недоступна, изменилась разметка, пустой ответ).
- Vue 3, Composition API; состояние загрузки и ошибки видно в интерфейсе.
- Понятная структура БД (миграции).

### Что оцениваем
Архитектуру и читаемость кода, обработку ошибок и крайних случаев, адекватность решения по парсингу, аккуратность работы с API между фронтом и бэком. Идеальный пиксель в вёрстке не важен.

- Ссылка на Git-репозиторий.
